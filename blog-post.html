What is the actual purpose of a Server?

I was confused about servers for the longest time, until I realized they are just about staying alive 24/7.

R

Rupesh

Frontend Developer

I was confused about servers for the longest time. Like, what's the actual purpose of a server in web development?

I kept hearing "you need to host your backend on a server" but I didn't really get why.

The "Click" Moment

Then it clicked. Servers = something that runs 24/7.

Whatever you put on a server stays live, all the time.

Here's the example that made it clear:

Take MongoDB. It's a physical database sitting somewhere. But when you host it on Atlas (which is a server), it becomes available online 24/7 for everyone.

The server is continuously running. Whatever you put on it? Also running live.

Why the Backend Needs a Server

Now here's where it all connects: Your website's frontend continuously needs data. That data comes from the backend.

So the backend needs to be online and active 24/7 so it can keep feeding data to the frontend whenever someone visits your site. That's where servers come into action.

Frontend = Static

It doesn't need to send data 24/7. You can host it on Netlify or Vercel, no problem.

Backend = Live 24/7

It has to be alive to answer requests. So we host it on servers like AWS, Railway, or Digital Ocean.

Database = Live 24/7

Also needs to be live. So we host it on Atlas (which is a server).

The Connected Ecosystem

Now everything is connected. They are all running 24/7, all talking to each other:

Frontend --> Netlify
Backend --> Railway (Server)
Database --> Atlas (Server)
